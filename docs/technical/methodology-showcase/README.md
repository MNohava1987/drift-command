# Methodology Showcase: Advanced Diagram-as-Code

This directory contains examples of three different professional methodologies for visualizing the **Drift Command** architecture.

| Methodology | Best For... | Key Feature |
|---|---|---|
| **[Advanced Mermaid](./1-Advanced-Mermaid.md)** | Quick, styled diagrams. | `classDef` and `subgraph` for a "Lucid" look. |
| **[D2 Declarative](./2-D2-Declarative.md)** | Large, complex systems. | Clean `{ }` syntax and superior auto-layout. |
| **[C4 Architecture](./3-C4-Architecture.md)** | Strategic documentation. | Layered "Zoom-In" approach (Context -> Container -> Component). |

## How to use this showcase:

1.  **Read the Markdown files:** Each file explains the "Strategy" and shows the "Diagram-as-Code" (the actual source you would write).
2.  **Try it out:** You can copy the code blocks into your favorite editor (VS Code, GitHub, or a D2/Mermaid online playground) to see them in action.
3.  **Choose your tool:**
    - Use **Mermaid** when you need something fast that renders natively in GitHub.
    - Use **D2** when your diagram gets so complex that Mermaid lines start crossing each other and looking like spaghetti.
    - Use **C4** when you need to explain your architecture to different groups (Stakeholders vs. Developers).
